# 接入指南 · 把你的应用接入星港（以 multillm-chat 为例）

> 目标：让用户在星港里打开你的应用，并**直接复用 ta 在星港「API 配置中心」配置的模型密钥**——用户换应用无需重配 key。
> 两种运行方式按应用形态二选一。

---

## 一、选运行方式

| 方式 | 适合 | 与平台通信 | 用平台密钥 |
|---|---|---|---|
| **嵌入沙箱 embedded** | 轻量、单视图的应用/工具 | iframe 内 `window.starport` postMessage SDK | `starport.ai.chat()`；首次自动弹"是否用平台密钥" |
| **新标签页 newtab** | 完整 web 应用（如 multillm-chat：多模型并排、流式、上传） | OAuth2 + REST API（服务器到服务器） | `POST /api/v1/ai/chat`（scope `gateway:llm`） |

multillm-chat 是完整 web 应用 → 用 **newtab + REST**。

---

## 二、在星港上架（开发者中心）

1. 打开 **开发者中心 → 创建应用**（类型选「应用」），得到 `client_id` 与一次性 `client_secret`。
2. 进应用编辑页：
   - **入口 URL** 填你的 multillm 部署地址（本地调试可填 `http://localhost:4000`）。
   - **运行方式** 选「新标签页」。
   - **运行要求 capabilities** 填 `llm:claude llm:openai`（声明它要用的 provider）。
3. 点「发布到商店」。

> 演示已帮你把 `multillm-chat` 设为：归你所有、newtab、`client_id=app_multillm`、入口 `http://localhost:4000`。在开发者中心可改入口为你的真实地址。

---

## 三、用户怎么用（端到端）

1. 用户在星港商店/库点 **启动** → 进 `/run/multillm-chat` → 因为是 newtab，显示「在新标签页打开应用」。
2. 点开 → 浏览器新开你的 multillm 网页。
3. multillm 引导用户 **「用星港账号登录」**（OAuth）→ 用户在星港授权页勾选 `gateway:llm`（即"允许用我的平台密钥"）。
4. 之后 multillm 的每次 AI 调用，走星港 Gateway 用用户在平台配的密钥 —— **multillm 自己不存、不碰用户明文 key**。

---

## 四、multillm-chat 要改的代码（具体）

multillm 现在用 `src/lib/providers/resolve.ts` 解析 key（用户 settings 或 env，支持 `apiKeyOverride`）。接入星港只需加一个「星港模式」：当用户用星港登录后，把 AI 调用改成调星港的代理。

### 1) 加「用星港登录」（OAuth2 授权码）
```ts
// 引导用户授权（前端跳转）
const authorize = `${STARPORT}/oauth/authorize?client_id=${CLIENT_ID}`
  + `&scope=identity,gateway:llm&redirect_uri=${encodeURIComponent(CALLBACK)}`;
location.href = authorize;

// 回调里用 code 换 access_token（multillm 后端，用 client_secret）
const r = await fetch(`${STARPORT}/api/v1/oauth/token`, {
  method: "POST", headers: { "content-type": "application/json" },
  body: JSON.stringify({ client_id: CLIENT_ID, client_secret: CLIENT_SECRET, code }),
});
const { access_token } = await r.json();   // 存进 multillm 的会话
```

### 2) 把 AI 调用改走星港 Gateway
multillm 现在大致是 `streamText({ model: resolve(provider, key), messages })`。
接入后，对「星港模式」的用户，改为调星港代理（用平台密钥）：
```ts
// 用星港平台密钥调用（multillm 后端）
const res = await fetch(`${STARPORT}/api/v1/ai/chat`, {
  method: "POST",
  headers: { "content-type": "application/json", authorization: `Bearer ${access_token}` },
  body: JSON.stringify({ provider: "anthropic", prompt }),  // 或 openai/gemini…
});
const { reply, usage } = await res.json();
```
- 想保留并排多模型：对每个模型并发调一次 `/api/v1/ai/chat`（provider 不同）。
- 想保留流式：第一步可先用非流式跑通；流式版可在 Gateway 加 SSE（后续增强）。
- `resolve.ts` 的 `apiKeyOverride` 那条分支天然适合挂"星港模式"开关。

### 3) 读用户身份（可选）
```ts
const me = await fetch(`${STARPORT}/api/v1/me`, { headers:{ authorization:`Bearer ${access_token}` }}).then(r=>r.json());
// { handle, name, avatarHue, level } —— 可用来显示"已用星港账号登录"
```

## 五、让 AI agent 自动接入（仿 Claude skills）

不想手写对接？把这两个 URL 交给 agent，它能自描述地理解并调用全部接口：
- 机器可读清单：`GET ${STARPORT}/api/v1/manifest`
- 技能说明：`GET ${STARPORT}/skill`（Claude-skill 风格 SKILL.md）

页面入口：**开发者中心 → 接入与开放 API**（`/developer/integrate`）。

---

## 六、关键保证
- 用户密钥只在星港服务端解密、调用大模型那一瞬存在；你的应用全程只拿到「调用结果」，**接触不到明文 key**。
- 用户一次授权 `gateway:llm`，**在所有接入应用间通用**——这正是"配置一次、全平台用"。
- 每次调用按真实 token 计入用户的用量看板，受日限额约束。
